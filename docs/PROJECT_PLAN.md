# Sticky Notes 桌面便箋專案規格

## 專案目標

開發 Windows 桌面便箋工具，提供類似 Microsoft Sticky Notes 的使用方式，並透過 Google Drive 在多台裝置之間同步。

核心原則：

- 每張便箋使用獨立 TXT 明文保存。
- 沒有網路時仍能正常使用桌面便箋。
- 桌面介面與雲端同步是獨立模組。
- 不依賴 Google Drive 電腦版。

## 技術方案

| 項目 | 技術 |
| --- | --- |
| 開發語言 | Python |
| 桌面介面 | PySide6 |
| 便箋內容 | TXT |
| 便箋資訊 | JSON |
| 雲端同步 | Google Drive API |
| Windows 打包 | PyInstaller |

## 系統架構

```text
PySide6 桌面介面
        ↓
本機 TXT 與 JSON 資料
        ↓
Google Drive API 同步模組
```

## 第一階段：本機桌面便箋

- 新增、編輯及刪除便箋。
- 開啟多張獨立便箋。
- 編輯後自動儲存。
- 支援拖曳、縮放、顏色、無邊框及置頂。
- 保存便箋與裝置限定的視窗狀態。
- 提供系統匣與選用的開機啟動功能。
- 寫入時使用暫存檔及原子取代。
- 刪除時先保留可復原紀錄。

## 第二階段：Google Drive 同步

- 第一次使用時透過瀏覽器完成 Google OAuth 授權。
- 在 Google Drive 建立專用資料夾。
- 上傳、下載及刪除便箋檔案。
- 記錄雲端檔案 ID、修改時間及同步狀態。
- 啟動時同步，本機修改後延遲同步。
- 網路或 API 失敗時保留本機資料並稍後重試。
- 兩台裝置同時修改時保留衝突副本，不靜默覆蓋。
- OAuth 憑證及 Token 不可提交至 Git。

## 建議資料格式

```text
data/
├─ notes/
│  ├─ <note-id>.txt
│  └─ <note-id>.txt
├─ metadata.json
└─ local-settings.json
```

視窗位置與尺寸屬於裝置限定設定，不應跨裝置同步。

## 開發順序

1. 完成本機 TXT 儲存與單一便箋視窗。
2. 加入多便箋、metadata 與視窗狀態。
3. 加入系統匣、刪除復原與開機啟動。
4. 建立 Google OAuth 與單檔上傳、下載。
5. 實作雙向同步、重試、刪除傳播及衝突副本。
6. 使用 PyInstaller 打包並在乾淨 Windows 環境驗證。

## 驗收條件

- 重啟程式後便箋內容保持一致。
- 異常中止不會損壞已完成儲存的便箋。
- 離線時能正常新增、修改及刪除便箋。
- 網路恢復後能補做同步。
- 同時修改時任一版本都不會直接遺失。
- TXT 能由一般文字編輯器直接開啟。
- 發布版不要求安裝 Python 或 Google Drive 電腦版。

## 暫不處理

- 手機原生 App
- 多人共同編輯
- 即時逐字同步
- 富文字、圖片及附件
- 自建後端伺服器
